If you want the maximum security of Option 1 but completely free, we can use a clever trick:

When a new user successfully creates an account, we hash their email (e.g., using SHA-256) and save it as a blank document in Firestore: registered_emails/<hashed_email>.
We set Firestore security rules so that people can only check if a specific hash exists, but they cannot list all the emails.
When someone clicks "Create Account", our app hashes the email they typed and asks Firestore: "Does this hash exist?" If yes, we show the "User already exists" error!
